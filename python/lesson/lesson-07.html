Lesson 7: Debugging and Exceptions

Read error traces and handle predictable failures.

Topics: try / except / traceback / logic checks

Quiz timer: 20 minutes (auto-submit on timeout)

Lesson Notes

Debugging is a process: read the error, isolate, fix, and retest.

Key Points

  • Traceback lines show where code failed.
  • Use try/except for predictable runtime errors.
  • NameError means something is undefined.
  • Add temporary prints to inspect values.

Example: safe integer input

try:
    age = int(input('Age: '))
except ValueError:
    print('Please enter a whole number')

Mini Practice

  • Cause a NameError intentionally and read the traceback.
  • Handle ZeroDivisionError in a simple division script.
